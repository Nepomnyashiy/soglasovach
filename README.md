# Согласовач (Soglasovach)

Платформа для автоматизации корпоративных процессов согласования. "Согласовач" предназначен для замены ручных, неэффективных и подверженных ошибкам процессов на управляемый, прозрачный и автоматизированный цифровой конвейер.

## Технологический Стек

*   **Бэкенд:** Python 3.11+, FastAPI, SQLAlchemy (async), Pydantic
*   **Фронтенд:** React 18+, TypeScript, Vite
*   **База данных:** PostgreSQL 15+
*   **Кэш/Очереди:** Redis 7+
*   **Администрирование БД:** pgAdmin 4
*   **Контейнеризация:** Docker, Docker Compose
*   **CI/CD:** GitHub Actions (будет настроено)

## Требования к Окружению

Для запуска проекта вам потребуется установленный Docker и Docker Compose.

## Быстрый Старт

1.  Клонируйте репозиторий:
    ```bash
    git clone <URL_ВАШЕГО_РЕПОЗИТОРИЯ>
    cd soglasovach
    ```
2.  Запустите все сервисы с помощью Docker Compose:
    ```bash
    docker compose up --build -d
    ```
    Это соберет образы (если необходимо) и запустит бэкенд, фронтенд, базу данных, Redis и pgAdmin в фоновом режиме.

## Доступные Сервисы

После успешного запуска Docker Compose, следующие сервисы будут доступны:

*   **Backend API:** `http://localhost:8000`
    *   Документация API (Swagger UI): `http://localhost:8000/docs`
    *   Документация API (ReDoc): `http://localhost:8000/redoc`
*   **Frontend UI:** `http://localhost:3000`
*   **pgAdmin:** `http://localhost:5050`
    *   Логин по умолчанию: `admin@soglasovach.com`
    *   Пароль по умолчанию: `admin`

## Разработка

*   **Hot-reloading:** Изменения в коде бэкенда (`./backend`) и фронтенда (`./frontend`) будут автоматически подхватываться и перезагружаться соответствующими сервисами благодаря монтированию томов в `docker-compose.yml`.
*   **Остановка сервисов:** Для остановки всех запущенных сервисов используйте:
    ```bash
    docker compose down
    ```
*   **Просмотр логов:** Для просмотра логов конкретного сервиса (например, бэкенда):
    ```bash
    docker compose logs -f backend
    ```
